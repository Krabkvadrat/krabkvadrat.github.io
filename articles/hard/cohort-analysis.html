Когортный анализ

Когортный анализ — метод изучения поведения пользователей, объединённых в группы (когорты). Чаще всего когорты формируются по дате первого взаимодействия с продуктом.

Зачем это нужно

Когортный анализ помогает ответить на важные вопросы: как меняется удержание пользователей со временем? Какие когорты показывают лучшие результаты? Влияют ли изменения в продукте на поведение пользователей? Окупаются ли затраты на привлечение клиентов?

На практике через когорты чаще всего анализируют retention и revenue. Часто можно услышать, что когорта «не созрела» — так говорят о когорте, которая ещё может продолжить наполняться.

Этапы когортного анализа

1
Определите цель анализа
Сформулируйте чёткий вопрос. Например: улучшилось ли удержание после редизайна приложения?
2
Выберите критерий формирования когорт
Дата регистрации (наиболее распространённый вариант), дата первой покупки, источник трафика, версия продукта при первом использовании, географическое положение.
3
Определите временные интервалы
Выберите период в зависимости от специфики бизнеса: дни, недели, месяцы или комбинированный вариант.
4
Решите, что измерять
Возвращение в приложение (DAU/WAU/MAU/retention rate), совершение покупки, использование ключевой функции, средний чек или выручка.
5
Соберите данные и постройте таблицу
Используйте SQL-запросы ниже. На выходе должна получиться матрица с когортами.
6
Интерпретируйте результаты
Горизонтальные паттерны: если одна когорта сильно отличается — ищите внешние факторы. Вертикальные: падение во всех когортах в одном периоде — возможно проблема с онбордингом. Диагональные: улучшение в поздних когортах — признак развития продукта.

Реализация в SQL

Практика на реальных данных: репозиторий с заданиями

SQL-запрос

WITH cohorts AS (
-- объединяем юзеров в когорты
    SELECT
        user_id,
        DATE_TRUNC('month', registration_date) AS cohort_month,
        registration_date
    FROM users
),
cohort_activity AS (
-- считаем сколько юзеров в когорте и сколько дожило до определённого момента
    SELECT
        c.cohort_month,
        COUNT(DISTINCT c.user_id) AS cohort_size,
        -- Week 2: вернулся в дни 8-14 (после первой недели)
        COUNT(DISTINCT CASE
            WHEN s.session_date BETWEEN c.registration_date + 8 AND c.registration_date + 14
            THEN c.user_id END) AS week_2,
        -- Week 3-4: вернулся в дни 15-30
        COUNT(DISTINCT CASE
            WHEN s.session_date BETWEEN c.registration_date + 15 AND c.registration_date + 30
            THEN c.user_id END) AS week_3_4,
        -- Month 2: вернулся в дни 31-60
        COUNT(DISTINCT CASE
            WHEN s.session_date BETWEEN c.registration_date + 31 AND c.registration_date + 60
            THEN c.user_id END) AS month_2
    FROM cohorts c
        LEFT JOIN user_sessions s
            ON c.user_id = s.user_id
        LEFT JOIN orders o
            ON c.user_id = o.user_id
                AND o.status = 'completed'
    GROUP BY c.cohort_month
)
-- финально считаем метрики
SELECT
    TO_CHAR(cohort_month, 'YYYY-MM') AS cohort,
    cohort_size,
    ROUND((week_2::decimal / cohort_size * 100), 2) AS week_2_retention,
    ROUND((week_3_4::decimal / cohort_size * 100), 2) AS week_3_4_retention,
    ROUND((month_2::decimal / cohort_size * 100), 2) AS month_2_retention
FROM cohort_activity
ORDER BY cohort_month

Cohort Analysis

Cohort analysis is a method for studying the behavior of groups of users (cohorts). Most commonly, cohorts are formed by the date of first interaction with the product.

Why it matters

Cohort analysis answers: how does user retention change over time? Which cohorts perform best? Do product changes affect user behavior? Are acquisition costs paying off?

Steps of cohort analysis

1
Define your goal
Formulate a clear question. For example: did retention improve after the app redesign?
2
Choose the cohort criterion
Registration date (most common), first purchase date, traffic source, product version on first use, geography.
3
Define time intervals
Days, weeks, months or a combination — depends on the business.
4
Decide what to measure
App returns (DAU/WAU/MAU/retention rate), purchases, key feature usage, average order value or revenue.
5
Build the cohort table
Use the SQL queries below. The output should be a retention matrix.
6
Interpret the results
Horizontal patterns: if one cohort stands out — look for external factors. Vertical: a drop across all cohorts in a single period — likely an onboarding or product issue. Diagonal: improvement in later cohorts — a sign the product is evolving.

SQL query

WITH cohorts AS (
-- group users into cohorts
    SELECT
        user_id,
        DATE_TRUNC('month', registration_date) AS cohort_month,
        registration_date
    FROM users
),
cohort_activity AS (
-- count cohort size and how many users returned at each period
    SELECT
        c.cohort_month,
        COUNT(DISTINCT c.user_id) AS cohort_size,
        -- Week 2: returned on days 8-14
        COUNT(DISTINCT CASE
            WHEN s.session_date BETWEEN c.registration_date + 8 AND c.registration_date + 14
            THEN c.user_id END) AS week_2,
        -- Week 3-4: returned on days 15-30
        COUNT(DISTINCT CASE
            WHEN s.session_date BETWEEN c.registration_date + 15 AND c.registration_date + 30
            THEN c.user_id END) AS week_3_4,
        -- Month 2: returned on days 31-60
        COUNT(DISTINCT CASE
            WHEN s.session_date BETWEEN c.registration_date + 31 AND c.registration_date + 60
            THEN c.user_id END) AS month_2
    FROM cohorts c
        LEFT JOIN user_sessions s
            ON c.user_id = s.user_id
        LEFT JOIN orders o
            ON c.user_id = o.user_id
                AND o.status = 'completed'
    GROUP BY c.cohort_month
)
-- final metrics
SELECT
    TO_CHAR(cohort_month, 'YYYY-MM') AS cohort,
    cohort_size,
    ROUND((week_2::decimal / cohort_size * 100), 2) AS week_2_retention,
    ROUND((week_3_4::decimal / cohort_size * 100), 2) AS week_3_4_retention,
    ROUND((month_2::decimal / cohort_size * 100), 2) AS month_2_retention
FROM cohort_activity
ORDER BY cohort_month
